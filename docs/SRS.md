## 1. 프로젝트 개요

본 프로젝트는 대학생 스터디 모집 과정에서 발생하는 목표 불일치와 모집 비효율 문제를 해결하기 위한 웹 기반 스터디 매칭 시스템 MVP이다.

기존의 스터디 모집은 커뮤니티 게시글, 단체 채팅방, 지인 추천 등에 의존하는 경우가 많아 사용자가 원하는 목표, 수준, 요일, 시간대가 일치하는 사람을 찾기 어렵다. 이로 인해 스터디 모집 이후에도 지속률이 낮아지고 관리 부담이 커지는 문제가 발생한다.

본 시스템은 사용자가 로그인 후 자신의 관심분야, 학습 목표, 학습 수준, 원하는 학습 요일과 시간을 입력하고, 이를 바탕으로 스터디를 생성하거나 조건에 맞는 스터디를 조회하여 참여 신청할 수 있도록 지원한다. 또한 스터디 생성자는 참여 신청을 승인 또는 거절할 수 있으며, 향후 자동 추천 및 매칭 기능으로 확장할 수 있도록 설계한다.

## 2. 문제 정의

대학생들은 스터디를 모집할 때 주로 커뮤니티 게시글이나 단체 채팅을 이용한다.

하지만 이 방식에는 다음과 같은 문제가 있다.

- 자신이 원하는 목표, 시간대가 일치하는 사람을 찾기 어렵다
- 관심 분야가 맞지 않는 경우가 많다
- 스터디 모집 및 관리 과정에 많은 시간과 비용을 투자해야 한다

따라서 사용자 프로필 기반으로 스터디를 생성하고 모집할 수 있는 
웹 기반 시스템이 필요하다.

## 3. 사용자

본 시스템의 주요 사용자는 다음과 같다.

1. 대학생
2. 자율 학습을 진행하는 일반 사용자

사용자는 자신의 관심분야와 학습 목표를 입력하고 
이를 기반으로 스터디를 생성하거나 모집에 참여할 수 있다.

## 4. 시스템 목표

본 시스템의 목표는 다음과 같다.

- 사용자 프로필 기반 스터디 모집 환경 제공
- 스터디 생성 및 모집 과정 단순화
- 스터디 관리 기능 제공
- 향후 자동 매칭 기능 확장을 위한 데이터 구조 설계

## 5. 기능 요구사항

### 회원 관리

- 사용자는 회원가입을 할 수 있다
- 사용자는 로그인할 수 있다

### 사용자 프로필

- 사용자는 자신의 관심분야를 입력할 수 있다
- 사용자는 학습 목표를 입력할 수 있다
- 사용자는 자신의 학습 수준(level)을 입력할 수 있다
- 사용자는 원하는 공부 요일을 입력할 수 있다
- 사용자는 원하는 공부 시간을 입력할 수 있다
- 프로필 정보는 데이터베이스에 저장된다

### 스터디 관리

- 사용자는 스터디를 생성할 수 있다
- 스터디 생성 시 제목, 분야(category), 모집 대상 수준(level), 설명, 모집 인원, 모집 요일, 모집 시간대를 입력할 수 있다
- 생성된 스터디는 목록에 표시된다
- 사용자는 스터디 목록에서 제목, 분야, 모집 인원, 모집 요일, 모집 시간대, 모집 상태를 확인할 수 있다
- 사용자는 조건에 맞는 스터디를 조회할 수 있다
- 사용자는 조건에 맞는 스터디에 참여 신청을 보낼 수 있다
- 스터디 생성자는 사용자의 참여 신청을 승인 및 거절할 수 있다
- 스터디 생성자는 스터디의 참여자 목록을 조회가 가능하다

### study_members 상태 전이 규칙

`study_members.status`는 스터디 참여 신청의 현재 상태를 나타내며, 다음 3가지 값을 가진다.

- `pending` : 참여 신청이 접수되었으나 아직 승인 또는 거절되지 않은 상태
- `approved` : 스터디 생성자가 참여 신청을 승인한 상태
- `rejected` : 스터디 생성자가 참여 신청을 거절한 상태

#### 상태 전이 규칙

- 참여 신청 생성 시 초기 상태는 `pending`
- `pending` 상태만 `approved` 또는 `rejected`로 변경 가능
- `approved` 또는 `rejected` 상태는 다시 `pending`으로 변경할 수 없다

## 6. 비기능 요구사항

- 시스템은 별도의 프로그램 설치 없이 웹 브라우저에서 실행 가능해야 한다.
- 인증되지 않은 사용자는 프로필 저장, 스터디 생성, 참여 신청 기능을 사용할 수 없다.
- 사용자 비밀번호는 평문이 아닌 암호화된 형태로 저장해야 한다.
- 회원가입 시 동일한 username은 중복 저장할 수 없어야 한다.
- 동일한 사용자는 같은 스터디에 중복으로 참여 신청할 수 없어야 한다.
- 모집 상태가 `closed`인 스터디에는 참여 신청 및 승인 처리를 할 수 없어야 한다.
- 스터디 승인 인원은 모집 인원(`max_members`)을 초과할 수 없어야 한다.
- 필수 입력값이 누락된 요청에 대해서는 오류 응답(400 Bad Request)을 제공해야 한다.
- 인증이 필요한 기능에 대해 인증되지 않은 요청은 오류 응답(401 Unauthorized)을 제공해야 한다.
- 중복 username 또는 중복 참여 신청과 같은 충돌 상황에 대해서는 오류 응답(409 Conflict)을 제공해야 한다.
- 참여 상태 변경 기능은 해당 스터디 생성자만 수행할 수 있어야 하며, 권한이 없는 요청에는 오류 응답(403 Forbidden)을 제공해야 한다.
- 시스템은 향후 자동 추천 및 매칭 기능이 추가될 수 있도록 사용자 프로필, 스터디 정보, 참여 정보를 분리된 구조로 관리해야 한다.

## 7. MVP 범위

본 프로젝트의 MVP(Minimum Viable Product) 범위는 다음과 같다.

- 회원가입 및 로그인
- 사용자 관심분야, 학습 목표, 학습 수준, 원하는 학습 요일 및 시간 입력
- 스터디 생성 기능
- 스터디 모집 및 목록 조회
- 스터디 참여 신청, 참여자 조회 및 참여 상태 변경

자동 매칭 기능은 향후 확장 기능으로 계획한다.

## 8. 향후 확장 계획

향후 시스템은 다음과 같은 기능으로 확장될 수 있다.

- 사용자 프로필 기반 자동 스터디 추천
- 스터디 일정 관리 기능
- 알림 기능
- 추천 스터디 시스템
- 스터디 채팅 기능
- 스터디 성과 공유 기능
