## 1. 프로젝트 개요

본 프로젝트는 대학생 스터디 모집 과정에서 발생하는 
목표 불일치와 모집 비효율 문제를 해결하기 위한 
웹 기반 스터디 매칭 시스템 MVP이다.

사용자는 로그인 후 자신의 관심분야와 원하는 시간대 학습목표를 입력하고
스터디를 생성하면 조건이 맞는 사람들과 모집을 진행 할 수 있다.

## 2. 문제 정의

대학생들은 스터디를 모집할 때 주로 커뮤니티 게시글이나 단체 채팅을 이용한다.

하지만 이 방식에는 다음과 같은 문제가 있다.

- 자신이 원하는 목표, 시간대가 일치하는 사람을 찾기 어렵다
- 관심 분야가 맞지 않는 경우가 많다
- 스터디 모집 및 관리 과정이 비효율적이다

따라서 사용자 프로필 기반으로 스터디를 생성하고 모집할 수 있는 
웹 기반 시스템이 필요하다.

## 3. 사용자

본 시스템의 주요 사용자는 다음과 같다.

1. 대학생
2. 자율 학습을 진행하는 일반 사용자

사용자는 자신의 관심분야와 학습 목표를 입력하고 
이를 기반으로 스터디를 생성하거나 모집에 참여할 수 있다.

## 4. 시스템 목표

본 시스템의 목표는 다음과 같다.

- 사용자 프로필 기반 스터디 모집 환경 제공
- 스터디 생성 및 모집 과정 단순화
- 스터디 관리 기능 제공
- 향후 자동 매칭 기능 확장을 위한 데이터 구조 설계

## 5. 기능 요구사항

### 회원 관리

- 사용자는 회원가입을 할 수 있다
- 사용자는 로그인할 수 있다

### 사용자 프로필

- 사용자는 자신의 관심분야를 입력할 수 있다
- 사용자는 학습 목표와 원하는 공부 시간을 입력할 수 있다
- 프로필 정보는 데이터베이스에 저장된다

### 스터디 관리

- 사용자는 스터디를 생성할 수 있다
- 스터디 생성 시 제목, 설명, 모집 인원을 입력할 수 있다
- 생성된 스터디는 목록에 표시된다
- 스터디 모집 상태를 확인할 수 있다

## 6. 비기능 요구사항

- 시스템은 웹 브라우저에서 실행 가능해야 한다
- 사용자 인증을 통해 접근을 제어한다
- 데이터는 데이터베이스에 안전하게 저장되어야 한다
- 향후 기능 확장을 고려한 구조로 설계한다

## 7. MVP 범위

본 프로젝트의 MVP(Minimum Viable Product) 범위는 다음과 같다.

- 회원가입 및 로그인
- 사용자 관심분야, 원하는 학습 시간 및 학습목표 입력
- 스터디 생성 기능
- 스터디 모집 및 목록 조회

자동 매칭 기능은 향후 확장 기능으로 계획한다.

## 8. 향후 확장 계획

향후 시스템은 다음과 같은 기능으로 확장될 수 있다.

- 사용자 프로필 기반 자동 스터디 추천
- 스터디 일정 관리 기능
- 알림 기능
- 추천 스터디 시스템
- 스터디 채팅 기능
- 스터디 성과 공유 기능
